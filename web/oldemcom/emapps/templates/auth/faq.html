Q: What is the EVE API?

A: The EVE API is an "application programming interface" provided by CCP to allow third-party software to retrieve information from the EVE server.

You can find more information here and here.

Q: Where do I find my API key?

A: You can get your user ID and API key here. Please make sure that you use your limited API key.

Q: I gave my API key my old corp. Will they not be able to authenticate themselves with it?

A: Yes, though I will notice the authentication in the logs and ask what is going on. When you have given your API key to anyone, and leave that organization, please do make sure they can't use it anymore. For this, you can just create a new API key on the EVE homepage. The same holds for when you leave EM, of course - please do disable your key then!

Q: How will you use my API key?

A: Your API key will be used solely and exclusively for forum authentication. This is an OOC promise, too, in case you have RP doubts :-).

Q: Is this secure?

A: As secure as it can get. Your API key is stored in a database that only Arkady Sadik has access to. The database is separate from the forum database, so even if the forums are compromised, the API keys will not be accessible. But of course, access to the full site this runs on will give out the API keys, too.

You can see what API calls are being made on the API log, and revoke your API key if something weird is happening.

Q: What could someone find out with the limited API key?

A: In case the API keys will ever get compromised (or Arkady turns to the dark side and breaks a promise), the following is a list of API calls that are possible with the limited API key, and what information they reveal.
Please do make sure that you only submit your limited API key, never the full key!

Again, the EM forums will only ever use the first API call listed here.

/account/Characters.xml.aspx
Returns a list of all characters on the account, together with their corporations. This is the API call used for forum authentication.
/char/CharacterSheet.xml.aspx
Most information from the character sheet, including skills and skill points, the wallet balance, implants in use, certificates, clone, and your corporation roles and titles.
/char/FacWarStats.xml.aspx
If the character is enlisted in Factional Warfare, this will return the faction the character is enlisted in, the current rank and the highest rank the character ever had attained, and how many kills and victory points he made yesterday, in the last week, and total.
/char/Medals.xml.aspx
A list of medals the character has, including non-public ones.
/char/SkillInTraining.xml.aspx
The skill currently in training.
/char/SkillQueue.xml.aspx
The full skill queue of the character.
/char/Standings.xml.aspx
All standings of the character with agents, NPC corporations and factions, as well as his private standings to characters and corporations.
/corp/CorporationSheet.xml.aspx
All sorts of information on the corporation. In addition to the normal "show info" information, this also lists hangar and wallet division names (but not balances), as well as the maximum member limit.
/corp/FacWarStats.xml.aspx
The same as the FacWarStats for the character, just for the corporation: Which faction it is enlisted in, and how many kills and victory points it made yesterday, in the last week, and total.
/corp/Medals.xml.aspx
A list of medals the corporation has created, their titles, descriptions, who created them, and when.

This is, to my knowledge, the full list of API calls possible with the limited API key.
